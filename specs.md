## 1) Project Description

Students will apply machine learning concepts and techniques learned in class to a real-world dataset.

### Goal

Develop a complete end-to-end understanding of:

- Formulating a supervised ML task
- Selecting appropriate models
- Training and evaluating models
- Interpreting and communicating results

### Learning Outcomes

- **LO1:** Define and differentiate machine learning techniques and approaches.
- **LO2:** Assess the impact of model design and parameter choices in context.
- **LO3:** Apply ML approaches to real-world tasks and communicate findings effectively.

---

## 2) Groupings

- Project must be done in **groups of 4**.
- If class size is not divisible by 4, instructor adjusts group sizes.
- **No group can exceed 4 members.**
- Students may choose their own group members.

---

## 3) Dataset and Project Proposal

Each group must propose a supervised ML project using a real-world dataset.

### Proposal Criteria

1. **Personally meaningful task**
   - Must be interesting to the group, not chosen solely for convenience/compliance.

2. **Relatively unexplored task**
   - Avoid heavily studied tasks commonly replicated in papers/blogs/tutorials.
   - Task need not be fully novel, but avoid direct replication.
   - **Toy datasets are not allowed.**

3. **Feasible data acquisition plan**
   - Prefer publicly available datasets.
   - If self-collecting data, show feasibility within project timeline.

### Suggested Data Sources

- Dataset-focused journals (e.g., _Data in Brief_, _Scientific Data_)
- Kaggle
- UCI Machine Learning Repository  
  (_Use caution: many datasets are already extensively studied._)

### Required Proposal Content

1. Supervised task in format: **“Predict Y based on X”**
2. Dataset source (or collection plan)
3. 1–2 sentence justification of interest to group members

### Approval Requirement

- Instructor approval is required.
- Approved task + dataset must be finalized by **January 30, 2026, 11:59 PM**.
- Follow instructor’s submission process for proposal approval.

---

## 4) Machine Learning Project Implementation

After approval, proceed with implementation.

### Recommended Workflow

1. **Data Preparation**
   - Collect and clean dataset.

2. **Exploratory Data Analysis (EDA)**
   - Study distributions, feature relationships, and data characteristics.
   - Include visualizations and statistical summaries.

3. **Data Preprocessing**
   - Perform transformations needed for modeling.
   - Conduct feature selection if needed.

4. **Model Selection and Training**
   - Select supervised models appropriate to the task.
   - Train with proper train/validation/test handling.
   - Avoid data leakage.

5. **Error Analysis and Tuning**
   - Analyze model errors.
   - Refine
